# Что уже есть на сайте

## Маршруты и flow

- Есть гостевой экран `/auth`.
- Есть защищённые страницы `/`, `/people`, `/profile`.
- Если пользователь не авторизован, приложение редиректит на `/auth`.
- Если пользователь уже вошёл, переход на `/auth` уводит на главную.
- Неизвестные маршруты редиректятся в зависимости от состояния сессии.

## Авторизация на фронте

- Есть форма входа.
- Есть форма регистрации.
- Проверяются обязательные поля при регистрации.
- Есть проверка на дубли по `email` и `username`.
- После входа и регистрации пользовательская сессия сохраняется в `localStorage`.
- Список зарегистрированных пользователей тоже хранится в `localStorage`.
- Есть выход из аккаунта.

## Главный экран

- Есть основной layout домашнего экрана.
- Есть верхняя панель с логотипом, текущим пользователем и кнопкой выхода.
- Есть нижняя навигация между `Карта`, `Люди`, `Вы`.
- Есть большой блок карты на главной странице.
- Есть боковые карточки со статусом и краткой сводкой.
- Есть нижний ряд информационных карточек.

## Карта

- Подключён `MapLibre`.
- Используется raster-слой OpenStreetMap.
- На карте отображается текущий пользователь.
- На карте отображаются выбранные точки из моковых данных.
- Есть кнопки управления картой от `MapLibre`.
- При наличии доступа к геолокации карта получает живую позицию пользователя через `navigator.geolocation.watchPosition`.
- При первом успешном получении геолокации карта центрируется на реальной позиции.
- Новые координаты сохраняются в состояние текущего пользователя на фронте.
- Если геолокация недоступна, используются стартовые координаты.

## Страница People

- Есть отдельная страница `/people`.
- Есть кнопка возврата назад.
- Есть заголовок и описание экрана.
- Есть поле поиска.
- Есть фильтр-плашки.
- Есть подготовленный блок под список людей.
- Есть empty-state, который показывает, что список пока пуст.

## Страница Profile

- Есть отдельная страница `/profile`.
- Есть карточка текущего пользователя.
- Показываются имя, username, инициалы, локация, город, bio, батарея, streak.
- Есть блок статистики.
- Есть дополнительные инфо-карточки.
- Есть блок `memory moments`.
- Есть форма редактирования профиля.
- Можно менять `name`, `username`, `city`, `place`, `bio`.
- Изменения профиля сохраняются в `localStorage`.

## UI и стили

- Проект собран на `Vite + React`.
- Используются `react-router-dom`, `tailwindcss`, `lucide-react`, `maplibre-gl`.
- Есть единый визуальный стиль: тёмный фон, градиенты, стеклянные панели, крупная типографика.
- Есть кастомные стили для карты, маркеров, подписей и контролов.
- Интерфейс адаптирован под мобильную и десктопную ширину на базовом уровне.

## Данные и структура

- Есть моковые данные для мест, карточек, статистики и profile-контента.
- Есть функции подготовки данных профиля для текущего пользователя.
- Есть заготовки под более сложный store друзей и карты, но они не подключены к текущему UI.

## Что можно считать готовым именно по фронту

- Базовый пользовательский flow: регистрация -> вход -> главная -> people -> profile -> выход.
- Рабочий клиентский роутинг.
- Рабочий каркас Zenly-подобного интерфейса.
- Рабочая карта с клиентской геолокацией текущего пользователя.
- Рабочее локальное редактирование профиля без участия бэкенда.

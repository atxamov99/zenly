# Чего не хватает на сайте по фронту

## Экран People

- Поиск сейчас только визуальный, без фильтрации.
- Фильтр-плашки сейчас только визуальные, без логики.
- Нет реального списка людей на экране.
- Нет карточек пользователей в списке.
- Нет состояний загрузки для списка.
- Нет отдельного пустого состояния для разных сценариев поиска и фильтрации.
- Нет действий на карточках: открыть, добавить, подписаться, пригласить.

## Главный экран и карта

- На карте нет отображения друзей или других пользователей.
- Нет выбора пользователя на карте.
- Нет фокуса карты на выбранном человеке.
- Нет переключения слоёв, режимов отображения или карты плотности.
- Нет панели с подробностями выбранной точки или пользователя.
- Нет клиентской истории перемещений, маршрутов или событий.
- Нет явного UI-состояния, когда геолокация запрещена браузером.
- Нет отдельного onboarding-сценария на разрешение геолокации.

## Профиль

- Нет загрузки или смены аватарки.
- Нет валидации `username` при редактировании профиля.
- Нет сообщений об успешном сохранении или ошибке сохранения.
- Нет кнопки отмены изменений.
- Нет отдельных настроек профиля и приватности.

## Общий frontend UX

- В интерфейсе есть проблема с кодировкой русского текста: часть строк отображается как `Р...`.
- Нет единой клиентской системы уведомлений, toast-сообщений или status feedback.
- Нет общей обработки состояний `loading`, `error`, `empty`, `offline` по приложению.
- Нет экрана `404` с нормальным UI, сейчас используется только редирект.
- Нет отдельного состояния "первый запуск" после регистрации.
- Нет нормального пользовательского сценария восстановления сессии с индикацией загрузки.

## Архитектура фронта

- Часть компонентов и стора подготовлена, но не используется в текущем приложении.
- Есть старые/черновые компоненты, которые стоит либо подключить, либо удалить.
- Нет единого описания frontend-состояний и структуры UI-данных для всех экранов.
- Нет разделения на переиспользуемые UI-patterns для форм, карточек и empty-state блоков.

## Качество и поддержка

- Нет фронтовых тестов.
- Нет проверки основных пользовательских сценариев.
- Нет визуальных регресс-тестов.
- Нет оптимизации bundle size, хотя в проекте уже есть тяжёлая карта.

## Что логично делать дальше именно по фронту

- Починить кодировку текстов.
- Доделать `People` до рабочего списка с поиском и фильтрами.
- Добавить нормальные UI-состояния для геолокации, загрузки и ошибок.
- Довести профиль до полноценного экрана настроек пользователя.
- Подчистить неиспользуемые компоненты и привести фронтовую структуру к одному рабочему сценарию.
